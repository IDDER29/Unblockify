What protects you

Trust, built into the architecture.

These aren't policies on a page. They're enforced in the database, the API layer, and the session — every request, every role.

Isolation

Strict multi-tenant isolation

Every record carries your organization's ID, and every query is filtered by the caller's org before it runs. A request for another tenant's data returns "not found" — never the data. Cross-tenant access is impossible by design, not by convention.

Accountability

Immutable audit log

Every login, invite, role change, and admin action is recorded as an append-only event. Owners get a built-in audit viewer and a one-click CSV export — so you always have a defensible record of who did what, and when.

Privacy

GDPR data tools

Data-subject rights are built in: a full organization export, a per-user data export, and right-to-erasure that removes a person's data on request. No tickets to support, no manual database surgery.

Access

Role-based access control

Three least-privilege roles — owner, instructor, and student. Each endpoint enforces the roles it allows server-side, so a student can't reach instructor tooling and an instructor can't reach owner analytics, no matter what the client sends.

In transit

Encryption in transit

Run Unblockify behind HTTPS and every request — credentials, session cookies, and student data — is encrypted on the wire. Sessions ride in a secure, httpOnly cookie that JavaScript can't read.

Residency

Local-first & data residency

Unblockify runs entirely on your own infrastructure with its own database — no third-party data sharing. The AI Teaching Assistant even works fully offline with a deterministic local fallback, so student work never has to leave your walls.

Accounts

Account protection

Authentication is rate-limited to slow credential-stuffing, passwords can be reset securely, and sessions are carried in signed, httpOnly cookies. The basics, done right, on every account.

Your data stays yours

Local-first, and self-hostable end to end.

Unblockify is a single Node service and a single SQLite database that live on hardware you control. There's no telemetry phoning home and no shared multi-customer cloud holding your students' work. Your data has exactly one residence: yours.

Because everything — including the AI assistant's offline fallback — runs locally, you can deploy Unblockify on a private network or fully air-gapped and it keeps working.
